SSM SVD Compression (Lab 1)

Цей проект досліджує стиснення моделей простору станів (SSM) за допомогою SVD-розкладу для прискорення інференсу та оцінки втрат у пам'яті моделі.

Архітектура системи

В основі моделі лежить матриця HiPPO-LegS, яка відповідає за запам'ятовування історії сигналу. Для стиснення використовується SVD-розклад (Singular Value Decomposition), який замінює повну матрицю (N x N) на наближення низького рангу r.

Реалізовано два типи інференсу:
- Full-rank: пряме множення матриці на вектор стану.
- Low-rank: оптимізоване множення через розкладені компоненти U, S та V. Це знижує обчислювальну складність з квадратичної O(N^2) до лінійної O(N*r) відносно розміру стану.

Експериментальна база включає тести на Associative Recall (пригадування за ключем) та Long-Range Regression (обробка довгих часових залежностей).

Результати дослідження

1. Вплив на пам'ять: При зменшенні рангу (r < 16 для N=64) модель починає стрімко втрачати здатність до "нескінченного" запам'ятовування. При r=2 або r=4 модель практично втрачає контекст минулого (кореляція падає до нуля).

2. Продуктивність на CPU: Для розмірності N=64 реальне прискорення складає 0.6x-0.9x. Це означає, що повноранговий метод працює навіть швидше через те, що NumPy ефективніше обробляє одну велику операцію, ніж декілька малих (U, S, V) в циклі Python.

3. Надійність: Статистичний аналіз (SEM та 95% ДІ) підтвердив, що деградація результатів при зменшенні рангу є закономірною та стабільною на великих вибірках.
